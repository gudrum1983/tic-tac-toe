# UI слой - (глупые React компоненты)

Так как приложение маленькое и ui простой, то он был вынесен отдельно как общий переиспользуемый слой.

**Папка структурирована по аналогии с Atomic Design** - так как он упрощает организацию компонентов.

    ui
     atoms/ - простые, односложные компоненты
     layouts/ -  шаблоны разположения/ лейауты (они же templates в Atomic Design)
     molecules/ - компоненты чуть посложнее состоящие из атомов
     organisms/ - более сложные компоненты, могут содержать в себе всё выше перечисленное 

**«Глупые/Презентационные»** (Dumb/Presentational) компоненты в React — это компоненты, отвечающие исключительно за
внешний вид и отображение данных; они не содержат бизнес-логику, запросы к API или управление состоянием (state), а
получают данные через props от «умных» компонентов и просто их показывают, делая приложение более модульным и легко
тестируемым. Они как марионетки, управляемые «умным» родителем, и их главная задача — отделение представления от логики.

**Ключевые характеристики «глупых» компонентов:**

- Презентационные: Их основная работа — отрисовать UI (кнопки, формы, списки).
- Получают данные через props: Все, что им нужно для отображения, приходит извне.
- Не имеют собственного состояния (state): Они «глупые», потому что не решают, какие данные загружать или как их
  обрабатывать.
- Не содержат бизнес-логику: Не делают запросы к серверу, не обрабатывают сложные события.
- Легко переиспользуются: Благодаря отсутствию логики, их можно использовать в разных частях приложения.

**Зачем они нужны:**

- Чистота кода: Разделяют логику и представление.
- Легкость тестирования: Проще тестировать презентационные компоненты, так как они предсказуемы.
- Переиспользуемость: Один и тот же компонент кнопки можно использовать с разными данными в разных местах.

**Правила импортов:** 
1. C наружи только по alias:  `@ui`
```jsx
import { Board, GameLayout, StatusPanel, Stepper } from '@ui';
``` 
2. Внутри между подпапками импорт через индекс подпапки `../../[NameGroupFolder]`
```jsx
`import { Square } from '../../atoms';` 
``` 
3. Внутри одной подпапки например Typography для Button через относительный путь до индекса компонента `../[NameFolderComponent]`
```jsx
import { Typography } from '../Typography';
```
